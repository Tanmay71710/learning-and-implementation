Parameters Processing Script

Created scripts/parameters.sh to demonstrate parameter handling:

#!/bin/bash
echo "=========================================="
echo "Parameter Processing Script"
echo "=========================================="
echo "Arguments received: $@"
echo "Number of arguments: $#"

if [ $# -eq 0 ]; then
    echo "No arguments provided"
else
    echo "Processing arguments:"
    for arg in "$@"; do
        echo "  - $arg"
    done
fi

Made executable: chmod +x scripts/parameters.sh

3
Long Running Task Script

Created scripts/long_task.sh to simulate long-running processes:

#!/bin/bash
echo "=========================================="
echo "Long Running Task Simulation"
echo "=========================================="
echo "Started at: $(date)"

total_steps=10
for i in $(seq 1 $total_steps); do
    echo "Progress: [$i/$total_steps] - $(date '+%H:%M:%S')"
    sleep 2
done

echo "Task completed successfully!"
echo "Finished at: $(date)"

Made executable: chmod +x scripts/long_task.sh

🚀 Usage Guide

Local Development Setup
  1. Install dependencies: pip install -r requirements.txt
  2. Copy environment template: cp .env.example .env
  3. Edit .env with your configuration
  4. Run application: python app.py
  5. Open browser: http://localhost:5000
Docker Deployment
  1. Build image: docker build -t script-execution-manager .
  2. Run container: docker run -p 5000:5000 script-execution-manager
  3. Or use docker-compose: docker-compose up -d
  4. Access application: http://localhost:5000
Jenkins Integration
  1. Follow JENKINS_SETUP.md for Jenkins configuration
  2. Create Jenkins job with SCRIPT_NAME parameter
  3. Configure build step to execute scripts
  4. Update .env with Jenkins credentials
  5. Test connectivity using health check endpoint
Tip: Start by testing without Jenkins using the local execution mode to verify the basic functionality before setting up Jenkins integration.
