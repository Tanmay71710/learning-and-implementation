A. Title and Overview

Jenkins CI/CD Server

Role in Project: Jenkins serves as the automation server for executing scripts, managing CI/CD pipelines, and providing build artifact storage for the Script Execution Manager.

Why Jenkins is Used

  • Mature Platform: Industry-standard with extensive plugin ecosystem
  • Flexible Pipeline: Declarative pipeline syntax for complex workflows
  • Extensible: Rich plugin ecosystem for integrations
  • Proven Reliability: Battle-tested in enterprise environments
  • Community Support: Large community and extensive documentation

GitOps Pipeline Position

Jenkins works in parallel with GitHub Actions in this project. While GitHub Actions handles the primary CI/CD pipeline, Jenkins provides script execution capabilities and serves as an alternative job execution backend for the Flask application.

Key Responsibilities

  • Execute scripts with configurable parameters
  • Provide job status monitoring and console output
  • Store build artifacts and logs
  • Integrate with version control systems
  • Support distributed builds with agents
  • Provide webhook integration for external triggers

D. Installation / Setup

Step 1: Deploy Jenkins in Kubernetes

# Create namespace
kubectl create namespace jenkins

# Deploy Jenkins using provided manifests
kubectl apply -f k8s/jenkins-deployment.yaml

# Or use Helm chart
helm repo add jenkins https://charts.jenkins.io
helm install jenkins jenkins/jenkins --namespace jenkins

Step 2: Configure Persistent Storage

# Create PersistentVolumeClaim for Jenkins
kubectl apply -f - <

Step 3: Configure Service and Ingress

# Expose Jenkins service
kubectl expose deployment jenkins --name=jenkins --port=8080 --namespace=jenkins --type=LoadBalancer

# Create ingress for external access
kubectl apply -f - <

Step 4: Initialize Jenkins

# Get initial admin password
kubectl exec -n jenkins deployment/jenkins -- cat /var/jenkins_home/secrets/initialAdminPassword

# Port-forward to access Jenkins UI
kubectl port-forward svc/jenkins -n jenkins 8080:8080

# Open browser to http://localhost:8080

Step 5: Configure Script Execution Job

# Via Jenkins UI:
# 1. Create new Freestyle project named "script-execution-job"
# 2. Add string parameter "SCRIPT_NAME"
# 3. Add build step to execute scripts from workspace
# 4. Enable "Trigger builds remotely" with authentication token
# 5. Configure post-build actions for artifact archiving

Step 6: Validate Setup

# Test Jenkins API access
curl -u admin:password http://jenkins.local:8080/api/json

# Test job triggering
curl -X POST -u admin:password http://jenkins.local:8080/job/script-execution-job/buildWithParameters?SCRIPT_NAME=test.sh
Tip: For production deployments, consider using Jenkins Configuration as Code (JCasC) to manage Jenkins configuration in version control.

F. Cookbook Recipes

Recipe 1: Create Basic Script Execution Job

Purpose: Set up a Jenkins job for executing scripts with parameters

Ingredients: Jenkins admin access, script workspace

Steps:

  1. Log in to Jenkins admin console
  2. Click "New Item"
  3. Enter job name: script-execution-job
  4. Select "Freestyle project"
  5. Add string parameter: SCRIPT_NAME
  6. Add build step: "Execute shell"
  7. Enter command: cd /path/to/scripts && ./$SCRIPT_NAME
  8. Configure post-build actions for log archiving

Expected Result: Jenkins job ready to execute scripts with parameters

Verification: Run test job with sample script

Recipe 2: Configure Webhook Integration

Purpose: Enable external triggering of Jenkins jobs via webhooks

Ingredients: Jenkins job configuration, authentication token

Steps:

  1. Navigate to job configuration
  2. Check "Trigger builds remotely"
  3. Set authentication token: my-secret-token
  4. Save configuration
  5. Test webhook: curl -X POST http://jenkins/job/script-execution-job/build?token=my-secret-token

Expected Result: Job can be triggered via HTTP POST with authentication

Verification: Test webhook from Flask application

Recipe 3: Set up Docker Agent

Purpose: Configure Jenkins to use Docker agents for isolated builds

Ingredients: Docker installed on agent, Jenkins configuration

Steps:

  1. Install Docker on agent machine
  2. Add Jenkins user to docker group
  3. Configure Jenkins agent in Jenkins UI
  4. Set agent to use Docker as template
  5. Configure cloud settings for Docker agents
  6. Test agent connectivity

Expected Result: Jenkins can spawn Docker agents for builds

Verification: Run pipeline with Docker agent

Recipe 4: Implement Declarative Pipeline

Purpose: Convert job to declarative pipeline with Jenkinsfile

Ingredients: Jenkinsfile, pipeline syntax knowledge

Steps:

  1. Create Jenkinsfile in repository
  2. Define pipeline structure with stages
  3. Add build steps and post-build actions
  4. Configure Pipeline job in Jenkins
  5. Set SCM to point to repository
  6. Test pipeline execution

Expected Result: Pipeline as code with version control

Verification: Trigger pipeline and check execution

Recipe 5: Configure Credential Management

Purpose: Securely manage credentials for external services

Ingredients: Jenkins credentials store, external service accounts

Steps:

  1. Navigate to Credentials → System → Global credentials
  2. Click "Add Credentials"
  3. Select credential type (username/password, secret text)
  4. Enter ID and description
  5. Add credentials
  6. Use in pipeline with withCredentials block

Expected Result: Credentials securely stored and injectable

Verification: Use credentials in test pipeline

G. Usage in This Project

Who Uses the Tool

  • Flask Application: Triggers Jenkins jobs for script execution
  • Developers: Monitor job execution and view console output
  • DevOps Engineers: Configure and maintain Jenkins infrastructure
  • CI/CD Pipeline: Alternative job execution backend

Event Triggers

  • Webhooks: Flask application triggers jobs via HTTP POST
  • Manual Triggers: Developers can trigger jobs via Jenkins UI
  • SCM Polling: Jobs can trigger on repository changes

Input/Output

  • Input: Script name, parameters, execution mode
  • Output: Job status, console output, exit code, build artifacts

System Integration

  • Flask Application: REST API integration for job triggering and monitoring
  • Kubernetes: Jenkins runs as containerized application
  • Git: Source code integration and pipeline as code
  • Artifactory: Optional artifact storage integration

Execution Flow

  1. User submits script execution request via Flask web interface
  2. Flask API triggers Jenkins job via webhook
  3. Jenkins executes script in configured workspace
  4. Job status and output are monitored by Flask application
  5. Results are returned to user via web interface

H. Best Practices

Naming Conventions

  • Use lowercase with hyphens for job names: script-execution-job
  • Use descriptive names for credentials: artifactory-admin
  • Use semantic versioning for Jenkins plugins
  • Use consistent naming for agent labels

Pipeline Structure

  • Use declarative pipeline syntax (Jenkinsfile)
  • Implement proper error handling and cleanup
  • Use stages for logical build phases
  • Implement parallel execution where possible
  • Use timeout directives to prevent hanging jobs

Credential Management

  • Never hardcode credentials in Jenkinsfile
  • Use Jenkins credentials store with proper IDs
  • Implement credential rotation policies
  • Use different credentials for different environments
  • Limit credential scope to minimum required

Security Hardening

  • Enable CSRF protection
  • Implement proper authentication and authorization
  • Use HTTPS/TLS for all communications
  • Regular security updates and patching
  • Implement network segmentation for agents

Resource Management

  • Set up resource limits for Jenkins master
  • Configure agent resource quotas
  • Implement workspace cleanup policies
  • Set up build discarding for old builds
  • Monitor resource usage and set up alerts

Monitoring and Logging

  • Enable comprehensive logging
  • Integrate with centralized logging systems
  • Monitor Jenkins master and agent health
  • Set up alerts for job failures
  • Track build metrics and trends

J. Troubleshooting

Problem: Job execution failures ▼

Symptoms: Jobs failing without clear error messages or hanging indefinitely

Likely Cause: Script errors, agent connectivity issues, resource exhaustion

How to Diagnose: Check console output, review agent logs, monitor resource usage

Fix: Fix script errors, restart agents, increase resource limits

Prevention: Implement proper error handling and resource monitoring

Problem: Agent connectivity issues ▼

Symptoms: Agents offline or unable to connect to master

Likely Cause: Network issues, incorrect agent configuration, firewall rules

How to Diagnose: Check agent logs, test network connectivity, verify agent configuration

Fix: Fix network issues, update agent configuration, adjust firewall rules

Prevention: Implement proper network configuration and monitoring

Problem: Webhook authentication failures ▼

Symptoms: Webhook triggers returning 403 or 401 errors

Likely Cause: Invalid tokens, CSRF protection, incorrect URL

How to Diagnose: Check token validity, review Jenkins security settings, test webhook URL

Fix: Update token, adjust security settings, correct webhook URL

Prevention: Use proper token management and security configuration

K. Q&A / FAQ

Q: Why is Jenkins used instead of only GitHub Actions?
A: Jenkins provides more flexibility for complex workflows, better agent management, and integration with existing infrastructure. It serves as an alternative execution backend.
Q: What happens if Jenkins job configuration is wrong?
A: Jobs will fail to execute or produce incorrect results. Always test job configuration with sample builds before production use.
Q: How do I validate Jenkins access configuration?
A: Test API access with curl: curl -u username:token http://jenkins/api/json, verify webhook authentication, test job triggering.
Q: How do I debug Jenkins pipeline failures?
A: Check pipeline logs in Jenkins UI, review console output, examine agent logs, and use pipeline replay for debugging.
Q: How do I rollback Jenkins configuration changes?
A: Use Jenkins configuration backup, revert changes via script, or reload previous configuration. Use JCasC for version-controlled configuration.
Q: What logs should I inspect for troubleshooting?
A: Jenkins master logs, agent logs, job console output, system logs, and application logs that interact with Jenkins.
Q: How do I secure Jenkins credentials?
A: Use Jenkins credentials store, environment variables, secret management systems, and implement proper access control.
Q: How does Jenkins behave in production?
A: In production, Jenkins should be configured with HA, proper security, monitoring, backup, and resource management. It handles higher load and provides reliability.
Q: How do I improve Jenkins performance?
A: Use distributed builds with agents, optimize pipeline steps, implement caching, increase resources, and tune JVM settings.
Q: What are the common limitations of Jenkins?
A: Configuration complexity, resource requirements, plugin compatibility issues, and maintenance overhead for large-scale deployments.
