4. Jenkins Pipeline Setup

Step 4.1: Jenkinsfile Creation

Created comprehensive Jenkinsfile with CI/CD pipeline:

pipeline {
    agent any
    
    environment {
        ARTIFACTORY_URL = 'http://10.118.244.255:9082'
        ARTIFACTORY_USERNAME = 'admin'
        ARTIFACTORY_PASSWORD = 'password'
        DOCKER_IMAGE = "script-execution-manager:${BUILD_NUMBER}"
        DOCKER_REPO = "10.118.244.255:9082/docker-local/script-execution-manager"
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Setup Environment') {
            steps {
                sh """
                    python3 -m venv venv
                    . venv/bin/activate
                    pip install -r requirements.txt
                """
            }
        }
        
        stage('Build Docker Image') {
            steps {
                sh """
                    docker build -t ${DOCKER_IMAGE} .
                    docker tag ${DOCKER_IMAGE} ${DOCKER_REPO}:${BUILD_NUMBER}
                """
            }
        }
        
        stage('Push to Artifactory') {
            steps {
                sh """
                    echo ${ARTIFACTORY_PASSWORD} | docker login ${ARTIFACTORY_URL} -u ${ARTIFACTORY_USERNAME} --password-stdin
                    docker push ${DOCKER_REPO}:${BUILD_NUMBER}
                """
            }
        }
        
        stage('Cleanup') {
            steps {
                sh """
                    docker logout ${ARTIFACTORY_URL}
                    docker rmi ${DOCKER_IMAGE} || true
                """
            }
        }
    }
}

Step 4.2: Pipeline Stages

Stage Description

Setup Environment Create Python virtual environment
Code Quality Run linting and formatting checks
Build Docker Image Build Docker image with build tags
Push to Artifactory Push to Artifactory Docker registry
Test Application Run application tests

8. External Access Configuration

Step 8.1: LoadBalancer Issue

❌ LoadBalancer service type didn't work for external access

Problem: LoadBalancer assigned VM IP (10.118.244.255) but wasn't accessible from Windows machine

Root Cause: LoadBalancer type requires cloud provider infrastructure; k3s mimics this but doesn't provide actual external load balancing

Step 8.2: Network Configuration Analysis

ip addr show
ens192: <BROADCAST,MULTICAST,UP,LOWER_UP>
    inet 10.118.244.255/22 brd 10.118.247.255 scope global ens192

VM Network: 10.118.244.255/22 on interface ens192

Step 8.3: NodePort Solution

Changed service type from LoadBalancer to NodePort:

apiVersion: v1
kind: Service
metadata:
  name: jenkins
  namespace: jenkins
spec:
  type: NodePort  # Changed from LoadBalancer
  selector:
    app: jenkins
  ports:
  - name: http
    port: 8080
    targetPort: 8080
    nodePort: 30080  # External access port
  - name: agent
    port: 50000
    targetPort: 50000
    nodePort: 30050

Step 8.4: Service Update

9. Troubleshooting Guide

Common Issues and Solutions

Issue 1: Artifactory Connection Reset

Symptoms: ERR_CONNECTION_RESET when accessing Artifactory from browser

Solution: Use SSH tunneling or access from server directly

ssh -L 9082:localhost:9082 user@server-ip

Issue 2: LoadBalancer Not Accessible Externally

Symptoms: LoadBalancer service not accessible from external machines

Solution: Use NodePort service type for local Kubernetes clusters

Issue 3: Kubernetes Pod Pending

Symptoms: Pods stuck in Pending state

Solution: Check PVC status and storage class availability

kubectl get pvc -n jenkins
kubectl get storageclass

Issue 4: Port Conflicts

Symptoms: Services fail to start due to port conflicts

Solution: Use different port mappings or stop conflicting services
