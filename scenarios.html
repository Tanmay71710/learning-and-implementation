Scenario 1: Basic GitOps Deployment Flow

Verify the basic GitOps workflow where changes in Git repository automatically trigger deployments through ArgoCD.

Pre-conditions

  • ArgoCD is installed and running
  • Kubernetes cluster is accessible
  • Git repository is connected to ArgoCD
  • Application is initially deployed

Test Steps

# Step 1: Make a simple configuration change cd environments/dev/k8s # Step 2: Modify the deployment replica count sed -i 's/replicas: 1/replicas: 2/' deployment-patch.yaml # Step 3: Commit and push changes git add deployment-patch.yaml git commit -m "Increase dev environment replicas to 2" git push origin develop

Verification Steps

  • Check ArgoCD application status: argocd app get script-execution-manager-dev
  • Verify sync status shows "Synced" within 2-3 minutes
  • Check Kubernetes deployment: kubectl get deployment -n script-execution-manager-dev
  • Verify replica count increased to 2
  • Check ArgoCD application logs for any errors

Expected Result

ArgoCD should detect the Git change, automatically sync the application, and update the Kubernetes deployment to 2 replicas without manual intervention.
