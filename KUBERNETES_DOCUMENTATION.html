🤖 GitHub Actions Workflow

1
Docker Build and Push Workflow

Created comprehensive GitHub Actions workflow docker-build-push-k8s.yml with:

Workflow Features:

  • Multi-platform builds: Supports amd64, arm64 architectures
  • Docker BuildKit: Advanced build caching and optimization
  • Smart tagging: Automatic tag generation (branch, SHA, semver)
  • Layer caching: Registry-based cache for faster builds
  • Build arguments: Metadata injection (timestamp, VCS ref)

Tagging Strategy:

Branch-based:  main, develop, feature-branch
SHA-based:    git commit SHA
Semver:      v1.0.0, v1.0.1, v1.0
Latest:      latest tag for default branch
2
Kubernetes Deployment Job

Automated Kubernetes deployment with:

  • Dynamic manifest updates: Updates image tags in manifests
  • Rolling updates: Zero-downtime deployments
  • Health verification: Post-deployment health checks
  • Rollback support: Automatic rollback on failure
  • Notifications: Email alerts on deployment status
3
Required GitHub Secrets

Configure these secrets in GitHub repository settings:

Secret Name Description Required
ARTIFACTORY_URL Artifactory server URL Yes
ARTIFACTORY_USERNAME Artifactory username Yes
ARTIFACTORY_PASSWORD Artifactory password/API key Yes
ARTIFACTORY_REPOSITORY Docker repository name Yes
KUBECONFIG Base64-encoded kubeconfig Yes (for K8s deployment)
EMAIL_* Email notification settings Optional

Workflow Triggers

The workflow triggers on:

  • Push to master, main, or develop branches
  • Tag pushes (v* semver tags)
  • Pull requests (build only, no deployment)
  • Manual dispatch with custom parameters

🐧 Artifactory Integration

1
Artifactory Docker Repository Setup

Step-by-Step Repository Creation:

  1. Log in to Artifactory as administrator
  2. Navigate to Administration → Repositories → Repositories → Local
  3. Click "New Local Repository"
  4. Select "Docker" as package type
  5. Set repository name (e.g., docker-local)
  6. Configure settings:
    • Tag retention: Keep last N tags
    • Trash retention: Delete after X days
    • Max unique tags: Limit tag versions
    • Access control: Set user/group permissions
  7. Click "Create" to finish
2
Generate Artifactory API Key
  1. Navigate to Administration → Profile → API Keys
  2. Click "Generate API Key"
  3. Set expiry date (recommended: 1 year)
  4. Copy and store the key securely
  5. Use this key in GitHub Secrets as ARTIFACTORY_PASSWORD
Important: Store API keys securely. They provide full access to Artifactory.
3
Configure Access Control

Repository Permissions:

  • Navigate to repository → Permissions
  • Add users/groups with appropriate access
  • Read/Deploy: Can pull images
  • Read/Annotate: Can pull and add metadata
  • Delete/Overwrite: Can push and delete images
  • Admin: Full repository management
4
Docker Client Configuration

Test Artifactory Connection:

# Login to Artifactory
docker login your-artifactory-url

# Tag image for Artifactory
docker tag script-execution-manager:latest \
  your-artifactory-url/docker-local/script-execution-manager:latest

# Push to Artifactory
docker push your-artifactory-url/docker-local/script-execution-manager:latest

# Verify image in Artifactory UI

☸️ Manual Kubernetes Deployment

1
Create Namespace
kubectl create namespace script-execution-manager
2
Create Image Pull Secret

Generate Secret Using Script:

# Set environment variables
export ARTIFACTORY_URL=your-artifactory-url
export ARTIFACTORY_USERNAME=your-username
export ARTIFACTORY_PASSWORD=your-password
export SECRET_NAME=artifactory-registry-secret
export NAMESPACE=script-execution-manager

# Run the secret generation script
./scripts/generate-artifactory-secret.sh

# Apply the generated secret
kubectl apply -f k8s/artifactory-registry-secret.yaml

Or Create Manually:

kubectl create secret docker-registry artifactory-registry-secret \
  --docker-server=your-artifactory-url \
  --docker-username=your-username \
  --docker-password=your-password \
  --namespace=script-execution-manager
3
Create ConfigMap

Apply the ConfigMap with application configuration:

kubectl apply -f k8s/configmap.yaml -n script-execution-manager
Note: ConfigMap contains non-sensitive configuration like Flask environment, job names, etc.
4
Create Secrets

First, update k8s/secrets.yaml with your actual values:

# Update these values in k8s/secrets.yaml:
database_url: "postgresql://user:password@postgres:5432/dbname"
jenkins_url: "http://jenkins-service:8080"
jenkins_username: "your-username"
jenkins_password: "your-password"
artifactory_url: "http://artifactory-service:8082"
artifactory_username: "your-username"
artifactory_password: "your-password"

Then apply the secrets:

kubectl apply -f k8s/secrets.yaml -n script-execution-manager
5
Deploy Application

Apply the deployment and service manifests:

# Update deployment with correct Artifactory URL
sed -i 's|ARTIFACTORY_URL|your-artifactory-url|g' k8s/deployment.yaml
sed -i 's|ARTIFACTORY_REPOSITORY|docker-local|g' k8s/deployment.yaml
sed -i 's|IMAGE_TAG|latest|g' k8s/deployment.yaml

# Apply deployment
kubectl apply -f k8s/deployment.yaml -n script-execution-manager

# Apply service
kubectl apply -f k8s/service.yaml -n script-execution-manager
6
Verify Deployment
# Check pods
kubectl get pods -n script-execution-manager -l app=script-execution-manager

# Check service
kubectl get svc -n script-execution-manager

# Get logs
kubectl logs -n script-execution-manager -l app=script-execution-manager --tail=50

# Describe deployment
kubectl describe deployment script-execution-manager -n script-execution-manager

# Check rollout status
kubectl rollout status deployment/script-execution-manager -n script-execution-manager

# Windows
choco install kubernetes-helm

# Verify installation
helm version
3
Create Production Values File

Create helm/script-execution-manager/values-production.yaml:

replicaCount: 5

image:
  repository: your-artifactory-url/docker-local/script-execution-manager
  tag: v1.0.0
  pullPolicy: Always

imagePullSecrets:
  - name: artifactory-registry-secret

service:
  type: LoadBalancer
  port: 80
  targetPort: 5000

resources:
  limits:
    cpu: 1000m
    memory: 1Gi
  requests:
    cpu: 500m
    memory: 512Mi

autoscaling:
  enabled: true
  minReplicas: 3
  maxReplicas: 10
  targetCPUUtilizationPercentage: 70

config:
  flaskEnv: production
  flaskDebug: "0"
  logLevel: WARNING

secrets:
  databaseUrl: postgresql://user:password@postgres:5432/dbname
  jenkinsUrl: http://jenkins:8080
  jenkinsUsername: your-username
  jenkinsPassword: your-password
  artifactoryUrl: http://artifactory:8082
  artifactoryUsername: your-username
  artifactoryPassword: your-password
4
Deploy with Helm
# Install the chart
helm install script-execution-manager helm/script-execution-manager \
  -f helm/script-execution-manager/values-production.yaml \
  -n script-execution-manager \
  --create-namespace

# Upgrade existing deployment
helm upgrade script-execution-manager helm/script-execution-manager \
  -f helm/script-execution-manager/values-production.yaml \
  -n script-execution-manager

# Uninstall the chart
helm uninstall script-execution-manager -n script-execution-manager
5

🔧 Troubleshooting

1
Pod Not Starting

Debug Steps:

  1. Check pod status: kubectl get pods -n script-execution-manager
  2. Describe pod: kubectl describe pod script-execution-manager-xxx -n script-execution-manager
  3. Check logs: kubectl logs script-execution-manager-xxx -n script-execution-manager
  4. Check events: kubectl get events -n script-execution-manager
2
Image Pull Errors

Common Issues:

  • Invalid credentials: Verify image pull secret
  • Wrong repository: Check Artifactory URL and repository name
  • Tag not found: Verify image tag exists in Artifactory
  • Network issues: Check cluster can access Artifactory
# Test image pull manually
kubectl run test-pod --image=your-artifactory-url/docker-local/script-execution-manager:latest \
  --image-pull-secret=artifactory-registry-secret -n script-execution-manager --rm -it -- sh
3
Deployment Failures

Rollback Strategies:

# Kubernetes rollback
kubectl rollout undo deployment/script-execution-manager -n script-execution-manager

# Rollback to specific revision
kubectl rollout undo deployment/script-execution-manager --to-revision=2 \
  -n script-execution-manager

# Helm rollback
helm rollback script-execution-manager -n script-execution-manager
4
Service Not Accessible

Debug Steps:

  1. Check service endpoints: kubectl get endpoints script-execution-manager -n script-execution-manager
  2. Describe service: kubectl describe svc script-execution-manager -n script-execution-manager
  3. Test connectivity: kubectl run test-pod --image=busybox -n script-execution-manager --rm -it -- wget -O- http://script-execution-manager/api/jenkins/health
  4. Check network policies: kubectl get networkpolicies -n script-execution-manager
