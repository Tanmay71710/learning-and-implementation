📧 Email Configuration

1
Gmail Setup

Step-by-Step Gmail Configuration:

  1. Enable 2-factor authentication on your Google account
  2. Go to Google Account settings → Security
  3. Enable "App passwords"
  4. Click "Select app" → choose "Other (Custom name)"
  5. Name it "GitHub Actions"
  6. Click "Generate"
  7. Copy the 16-character app password
  8. Use this as EMAIL_PASSWORD in GitHub Secrets
Important: Gmail requires app passwords for third-party access. Your regular password will not work.
2
Outlook/Office 365 Setup

Outlook Configuration:

EMAIL_SERVER: smtp.office365.com
EMAIL_PORT: 587
EMAIL_USERNAME: your-email@outlook.com
EMAIL_PASSWORD: your-regular-password
  • Use your regular Outlook password
  • Ensure SMTP access is enabled in Outlook settings
  • Port 587 for TLS, port 465 for SSL

Configure Multiple Recipients

🔧 Troubleshooting

1
Email Not Sending

Common Issues and Solutions:

  • Incorrect SMTP credentials: Verify username and password/app password
  • Wrong SMTP server: Check provider's SMTP server address
  • Port blocking: Try alternative ports (587 for TLS, 465 for SSL)
  • Firewall issues: Ensure GitHub Actions can access SMTP server
  • Rate limiting: Check email provider's sending limits
2
Workflow Failing

Debugging Steps:

  1. Check GitHub Actions logs for specific error messages
  2. Verify all dependencies are in requirements.txt
  3. Ensure Python version compatibility
  4. Check for syntax errors in Python files
  5. Review workflow YAML syntax
  6. Verify GitHub Actions permissions
3
Pylint Score Issues

Solutions:

  • Review Pylint messages in workflow logs
  • Fix reported code issues
  • Adjust .pylintrc configuration if needed
  • Consider disabling certain checks if not applicable
  • Focus on high-priority issues first
4
Permission Issues

Check:

  • GitHub Actions has permission to send emails
  • Repository Actions settings are properly configured
  • Secrets are properly configured at repository level
  • Workflow has necessary permissions
5
Gmail App Password Issues

Gmail-Specific Troubleshooting:

  • Ensure 2-factor authentication is enabled
  • Generate new app password if current one doesn't work
  • Check Google account security settings
  • Verify "Less secure apps" is not blocking access
  • Try using app password instead of regular password

📈 Advanced Configuration

Scheduled Runs

Add scheduled runs for regular quality checks:

on:
  schedule:
    - cron: '0 9 * * 1'  # Every Monday at 9 AM
    - cron: '0 0 * * *'  # Every day at midnight

Conditional Email Sending

Send emails only on failures:

- name: Send Email Notification
  if: failure()  # Only send on failure
  uses: dawidd6/action-send-mail@v3

Branch Protection Rules

Require checks before merging:

  1. Go to Settings → Branches
  2. Add branch protection rule
  3. Require status checks to pass
  4. Select "Code Quality Check and Notification"
