📋 GitOps Overview

This guide implements ArgoCD as the source of truth for GitOps deployment, providing:

  • Git as Source of Truth: Git repository contains all deployment configuration
  • Automated Synchronization: ArgoCD syncs Git changes to Kubernetes automatically
  • Multi-Environment Support: Dev, staging, and production environments
  • Rollback Capabilities: Easy rollback via Git revert
  • Health Monitoring: Application health checks and monitoring
  • Declarative Configuration: Infrastructure as code approach
  • RBAC Integration: Role-based access control
GitOps Status: ✅ Complete and Production Ready

1
Create Namespace

Create the ArgoCD namespace:

kubectl apply -f argocd/namespace.yaml
2
Install ArgoCD

Install ArgoCD using official manifests:

kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

Or use the custom configuration:

kubectl apply -f argocd/argocd-cm.yaml -n argocd
kubectl apply -f argocd/argocd-rbac-cm.yaml -n argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
3
Access ArgoCD UI

Port-forward to access the ArgoCD dashboard:

kubectl port-forward svc/argocd-server -n argocd 8080:44343

Get the initial admin password:

kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath='{.data.password}' | base64 -d

Access at: http://localhost:8080

1
Create ArgoCD Project

Apply the project configuration:

kubectl apply -f argocd/project.yaml

Project Features:

  • Source Repository: GitHub repository with Kubernetes manifests
  • Destinations: Multiple Kubernetes clusters (dev, staging, production)
  • Resource Whitelisting: Controlled resource types
  • RBAC: Admin and developer roles
  • Sync Windows: Automated sync policies
2
RBAC Configuration

The project includes role-based access control:

Role Capabilities
Admin Full access to all resources, modify project configuration
Developer Application management, view and sync apps, execute commands
1
Create Application

Apply the application manifest:

kubectl apply -f argocd/application.yaml

Application Configuration:

  • Source: Git repository and path
  • Destination: Kubernetes cluster and namespace
  • Sync Policy: Automated with pruning and self-healing
  • Health Checks: Application monitoring enabled
  • Revision History: Track deployment history
2
Sync Policy Details

The application uses the following sync policy:

syncPolicy:
  automated:
    prune: true        # Remove resources not in Git
    selfHeal: true      # Auto-revert manual changes
  syncOptions:
  - CreateNamespace=true  # Auto-create namespaces
  - ServerSideApply=true  # Server-side apply for better merge
1
App of Apps Pattern

Use the App of Apps for managing multiple environments:

kubectl apply -f argocd/app-of-apps.yaml

Environment Configuration:

Environment Branch Namespace Cluster
Production master script-execution-manager-prod production-cluster
Staging develop script-execution-manager-staging
Development develop deployment in-cluster
2
Branch Strategy
  • master: Production deployments
  • develop: Staging and development deployments
  • feature/*: Development and testing only
  • tags (v*): Production releases

GitOps Workflow

The `.github/workflows/argocd-gitops.yml` workflow integrates with ArgoCD:

Workflow Features:

  • Automated ArgoCD CLI installation
  • Environment-based sync triggers
  • Health check verification
  • Email notifications
  • Manual dispatch support
2
Workflow Triggers
  • Push to master → Production sync
  • Push to develop → Staging sync
  • Push to feature/* → Development sync
  • Tag pushes → Production sync with version
  • Manual dispatch with environment selection
3
Required Secrets

Add these to GitHub Secrets:

Secret Name Description Required
ARGOCD_URL ArgoCD server URL Yes
ARGOCD_USERNAME ArgoCD username Yes
ARGOCD_PASSWORD ArgoCD password Yes
1
Development Workflow
  1. Make changes to Kubernetes manifests in k8s/ directory
  2. Commit changes with descriptive message
  3. Push to Git repository
  4. ArgoCD detects changes automatically
  5. ArgoCD syncs changes to Kubernetes
  6. Application rolls out with new configuration
  7. Health checks verify deployment success
2
Deployment Workflow
  1. Make code changes
    2. >GitHub Actions builds Docker image
  2. Pushes image to Artifactory
  3. Updates image tag in Git manifests
  4. Commits and pushes changes to Git
  5. ArgoCD detects manifest changes
  6. ArgoCD syncs to Kubernetes
  7. Kubernetes rolls out new deployment
  8. Health checks verify deployment
3
Rollback Workflow

If deployment fails:

  1. Revert problematic changes in Git
  2. Push revert to Git repository
  3. ArgoCD detects rollback automatically
  4. Application rolls back to previous version
  5. Health checks verify rollback success

Application Health

Monitoring Commands
# Check application status

argocd app history script-execution-manager-prod

# Watch real-time sync status
argocd app watch script-execution-manager-prod

# Get application details
argocd app get script-execution-manager-prod -o json
