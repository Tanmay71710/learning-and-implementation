1. Basic Application Deployment

Use Case

Deploy a simple nginx application using ArgoCD with auto-sync enabled.

Application Manifest

apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: nginx-demo
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/your-org/argocd-demo.git
    targetRevision: main
    path: nginx-demo
  destination:
    server: https://kubernetes.default.svc
    namespace: nginx-demo
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
    - CreateNamespace=true

Kubernetes Deployment (in Git repo)

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: nginx-demo
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
  namespace: nginx-demo
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
  type: LoadBalancer

Deployment Commands

argocd app create nginx-demo --file nginx-app.yaml
argocd app get nginx-demo
argocd app sync nginx-demo
argocd app list
Tip: Always start with manual sync operations to understand the behavior before enabling automated sync.

Rollback Commands

argocd app sync rollback-app --revision
argocd app rollback rollback-app --hard

Manual Rollback Procedure

# View application history
argocd app history myapp

# Rollback to specific revision
argocd app sync myapp --revision abc123def456

# Rollback using git revert
git revert HEAD
git push origin main
argocd app sync myapp

# Emergency rollback - disable auto-sync first
argocd app set myapp --sync-policy none
argocd app rollback myapp --revision 2
# Fix the issue
argocd app set myapp --sync-policy automated
Warning: Always disable automated sync before performing emergency rollbacks to prevent immediate re-sync of broken configurations.

10. Progressive Delivery with Argo Rollouts

Rollout Resource

apiVersion: argoproj.io/v1alpha1

metadata:
  name: canary-demo
  namespace: canary
spec:
  replicas: 5
  selector:
    matchLabels:
      app: canary-demo
  template:
    metadata:
      labels:
        app: canary-demo
    spec:
      containers:
      - name: canary-demo
        image: argoproj/rollouts-demo:blue
        ports:
        - containerPort: 8080
  strategy:
    canary:
      steps:
      - setWeight: 20
      - pause: {duration: 10m}
      - setWeight: 40
      - pause: {duration: 10m}
      - setWeight: 60
      - pause: {duration: 10m}
      - setWeight: 80
      - pause: {duration: 10m}
      analysis:
        templates:
        - templateName: success-rate
        args:
        - name: service-name
          value: canary-demo

          sum(rate(http_requests_total{namespace="canary",service="canary-demo",status!~"5.."}[5m]))
