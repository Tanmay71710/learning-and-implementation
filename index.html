Project Overview

Script Execution Manager is a comprehensive GitOps project that demonstrates modern DevOps practices for deploying and managing a Flask-based web application. This project showcases a complete CI/CD pipeline with automated code quality checks, artifact management, and GitOps deployment strategies.

Key Features

  • Web-based Interface: Modern Flask application for script execution and monitoring
  • Jenkins Integration: Trigger and monitor Jenkins jobs with fallback to local execution
  • Artifact Management: JFrog Artifactory integration for Docker image storage
  • GitOps Workflow: Argo CD as the source of truth for Kubernetes deployments
  • Automated CI/CD: GitHub Actions for code quality, image building, and deployment
  • Multi-Environment Support: Development, staging, and production configurations

Quick Reference

Project Configuration

  • Repository: https://github.com/Tanmay71710/learning-and-implementation
  • Flask App Name: script-execution-manager
  • Artifact Repository: docker-local in Artifactory
  • Container Registry: http://10.118.244.255:30082/artifactory
  • Jenkins Pipeline: Declarative pipeline with webhook integration
  • Kubernetes Cluster: k3s (on-prem)
  • Namespaces: script-execution-manager, nginx-demo, jenkins, artifactory
  • Argo CD Applications: script-execution-manager, nginx-demo
  • Deployment Strategy: Rolling update
  • Ingress Controller: Traefik (built into k3s)
  • Secrets Management: Kubernetes Secrets

Environment Variables

  • FLASK_ENV: development / production
  • DATABASE_URL: sqlite:///script_manager.db
  • JENKINS_URL: http://jenkins-service:8080
  • ARTIFACTORY_URL: http://artifactory-service:8082
  • ARGOCD_URL: http://argocd-server.argocd.svc.cluster.local

Known Limitations

  • ApplicationSet CRD not installed (annotation size limitation in k3s)
  • Placeholder image references in development manifests
  • LoadBalancer external IP pending (k3s limitation)
  • Basic authentication only (OIDC configured but not fully tested)

Deployment Lifecycle

1. DEVELOPMENT
   └─> Developer pushes code to feature branch
   └─> GitHub Actions runs code quality checks
   └─> PR created for review

2. STAGING
   └─> Code merged to develop branch
   └─> GitHub Actions builds Docker image
   └─> Image pushed to Artifactory
   └─> Argo CD syncs to staging namespace
   └─> Automated testing and validation

3. PRODUCTION
   └─> Code merged to master branch
   └─> GitHub Actions builds production image
   └─> Image tagged and pushed to Artifactory
   └─> Argo CD syncs to production namespace
   └─> Rolling update with health checks
   └─> Monitoring and alerting

Getting Started

To get started with this project, follow these steps:

  1. Clone the repository: git clone https://github.com/Tanmay71710/learning-and-implementation.git
  2. Review the documentation: Start with the Flask Cookbook to understand the application structure
  3. Set up local environment: Configure environment variables using .env.example
  4. Install dependencies: pip install -r requirements.txt
  5. Run locally: python app.py
  6. Explore each technology: Use the individual cookbooks to set up each component
