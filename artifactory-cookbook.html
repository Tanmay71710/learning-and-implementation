A. Title and Overview

JFrog Artifactory

Role in Project: Artifactory serves as the universal artifact repository for Docker images, build artifacts, and dependencies in the Script Execution Manager GitOps pipeline.

Why Artifactory is Used

  • Universal Repository: Supports Docker, Maven, npm, and many other package formats
  • High Availability: Enterprise-grade repository with redundancy and replication
  • Security: Fine-grained access control and vulnerability scanning
  • Integration: Seamless integration with CI/CD tools like Jenkins and GitHub Actions
  • Performance: CDN capabilities and optimized artifact storage

GitOps Pipeline Position

Artifactory sits between the CI pipeline (GitHub Actions/Jenkins) and the deployment environment. It stores Docker images that are built during the CI process and pulled by Kubernetes during deployment.

Key Responsibilities

  • Docker image storage and distribution
  • Build artifact management
  • Dependency proxy for external repositories
  • Access control and security scanning
  • Replication and high availability

D. Installation / Setup

Step 1: Deploy Artifactory in Kubernetes

# Create namespace
kubectl create namespace artifactory

# Deploy Artifactory using Helm (recommended)
helm repo add jfrog https://charts.jfrog.io
helm install artifactory jfrog/artifactory --namespace artifactory

# Or use provided Kubernetes manifests
kubectl apply -f k8s/artifactory-deployment.yaml

Step 2: Configure Persistent Storage

# Create PersistentVolumeClaim
kubectl apply -f - <

Step 3: Configure Service and Ingress

# Expose Artifactory service
kubectl expose deployment artifactory --name=artifactory --port=8081 --namespace=artifactory

# Create ingress for external access
kubectl apply -f - <

Step 4: Initialize Artifactory

# Access Artifactory UI
kubectl port-forward svc/artifactory -n artifactory 8082:8081

# Open browser to http://localhost:8082
# Default credentials: admin / password

Step 5: Create Docker Repository

# Via Artifactory UI:
# 1. Navigate to Administration → Repositories → Repositories → Local
# 2. Click "New Local Repository"
# 3. Select "Docker"
# 4. Name: docker-local
# 5. Click "Create"

Step 6: Validate Setup

# Test Docker registry access
docker login artifactory.local:8082
# Use admin/password credentials

# Pull test image
docker pull nginx:latest
docker tag nginx:latest artifactory.local:8082/docker-local/nginx:latest
docker push artifactory.local:8082/docker-local/nginx:latest
Tip: For production deployments, use Artifactory High Availability (HA) setup with multiple nodes and shared storage for redundancy and performance.

  • Not setting storage quotas leading to disk space issues
  • Incorrect repository type for use case
  • Missing HTTPS/TLS configuration for production
  • Not implementing proper cleanup policies

F. Cookbook Recipes

Recipe 1: Create Docker Local Repository

Purpose: Set up a Docker repository for container image storage

Ingredients: Artifactory admin access, repository configuration

Steps:

  1. Log in to Artifactory admin console
  2. Navigate to Administration → Repositories
  3. Click "New Local Repository"
  4. Select "Docker" as package type
  5. Enter repository name: docker-local
  6. Configure base URL and port (default: 8082)
  7. Click "Create"

Expected Result: Docker repository ready for image push/pull

Verification: Test with docker login artifactory:8082

Recipe 2: Configure CI/CD Service Account

Purpose: Create service account for Jenkins/GitHub Actions integration

Ingredients: Artifactory admin access, user management

Steps:

  1. Navigate to Administration → Users & Groups
  2. Click "New User"
  3. Enter username: ci-cd-service
  4. Generate strong password
  5. Add to appropriate groups
  6. Create API key for automation
  7. Save credentials securely

Expected Result: Service account with API key for CI/CD

Verification: Test login with service account credentials

Recipe 3: Set up Repository Replication

Purpose: Configure replication for high availability

Ingredients: Multiple Artifactory instances, network connectivity

Steps:

  1. Configure remote Artifactory instance
  2. Navigate to Administration → Replication
  3. Click "New Replication"
  4. Select source and target repositories
  5. Configure replication frequency and event types
  6. Test replication with sample artifact

Expected Result: Artifacts automatically replicated between instances

Verification: Push artifact to source, verify in target

Recipe 4: Configure Storage Cleanup

Purpose: Implement automatic cleanup of old artifacts

Ingredients: Artifactory admin access, cron expressions

Steps:

  1. Navigate to Administration → Maintenance
  2. Click "New Cleanup Policy"
  3. Select repositories to clean up
  4. Configure retention rules (e.g., keep last 10 versions)
  5. Set schedule using cron expression
  6. Enable policy

Expected Result: Old artifacts automatically cleaned up

Verification: Monitor storage usage over time

Recipe 5: Integrate with GitHub Actions

Purpose: Configure Artifactory for GitHub Actions CI/CD

Ingredients: GitHub repository, Artifactory credentials

Steps:

  1. Add Artifactory credentials to GitHub Secrets
  2. Configure Docker login in GitHub Actions workflow
  3. Set up build and push steps in workflow
  4. Configure image tagging strategy
  5. Test workflow with sample build

Expected Result: GitHub Actions can push images to Artifactory

Verification: Trigger workflow and check Artifactory for new images

H. Best Practices

Naming Conventions

  • Use lowercase with hyphens for repository names: docker-local
  • Use semantic versioning for Docker tags: v1.0.0, v1.1.0
  • Use descriptive names for permission targets: ci-cd-permissions
  • Use consistent naming across environments: docker-dev, docker-prod

Repository Organization

  • Separate repositories for different environments (dev, staging, prod)
  • Use virtual repositories to aggregate multiple local repositories
  • Implement repository hierarchies for complex projects
  • Use remote repositories for external dependencies

Access Control

  • Implement principle of least privilege for service accounts
  • Use groups to manage permissions efficiently
  • Regular audit of user permissions and access logs
  • Integrate with LDAP/Active Directory for enterprise environments

Security Hardening

  • Enable HTTPS/TLS for all Artifactory access
  • Implement vulnerability scanning for Docker images
  • Use API keys instead of passwords for automation
  • Regular security updates and patching
  • Implement IP whitelist for admin access

Storage Management

  • Set up storage quotas per repository
  • Implement automated cleanup policies
  • Monitor storage usage and set up alerts
  • Use storage class with appropriate performance characteristics
  • Regular backup of Artifactory configuration and metadata

Performance Optimization

  • Use CDN for distribution to geographically dispersed teams
  • Implement repository replication for high availability
  • Optimize storage layout for frequently accessed artifacts

  • Monitor and tune Artifactory JVM settings

Monitoring and Logging

  • Regular review of system logs and access patterns

J. Troubleshooting

Problem: Docker push authentication failures ▼

Symptoms: "unauthorized: authentication required" or "no basic auth credentials"

Likely Cause: Invalid credentials, expired token, or incorrect repository URL

How to Diagnose: Test login manually, check credential validity, verify repository URL

Fix: Update credentials, regenerate API key, correct repository URL

Prevention: Use service accounts with API keys, regular credential rotation

Problem: Slow Docker image pulls ▼

Symptoms: Docker pull operations take excessive time

Likely Cause: Network issues, large image sizes, lack of CDN, or storage performance

How to Diagnose: Check network connectivity, monitor Artifactory performance, analyze image sizes

Fix: Optimize image sizes, implement CDN, improve storage performance

Prevention: Use multi-stage builds, implement layer caching, monitor performance

Problem: Storage space exhaustion ▼

Symptoms: Artifactory stops accepting new artifacts, disk space warnings

Likely Cause: No cleanup policies, large artifacts, lack of storage quotas

How to Diagnose: Check storage usage, analyze artifact sizes, review cleanup policies

Fix: Implement cleanup policies, delete old artifacts, increase storage capacity

Prevention: Set storage quotas, implement automated cleanup, monitor usage

Problem: Repository not accessible from Kubernetes ▼

Symptoms: Kubernetes pods cannot pull images from Artifactory

Likely Cause: Network policies, DNS issues, imagePullSecrets not configured

How to Diagnose: Check pod logs, test DNS resolution, verify imagePullSecrets

Fix: Configure imagePullSecrets, update network policies, fix DNS configuration

Prevention: Test connectivity during setup, use proper service discovery

K. Q&A / FAQ

Q: Why is Artifactory needed instead of Docker Hub?
A: Artifactory provides better security, access control, integration with CI/CD tools, and support for multiple artifact types in a single platform.
Q: What happens if Artifactory configuration is wrong?
A: CI/CD pipelines will fail, deployments will be unable to pull images, and development workflow will be disrupted. Always validate configuration with test operations.
Q: How do I validate Artifactory access configuration?
A: Test Docker login: docker login artifactory-url, test repository access via API, and verify permissions with limited user accounts.
Q: How do I debug Docker push failures?
A: Check Docker daemon logs, verify Artifactory credentials, test network connectivity, and examine Artifactory access logs for detailed error information.
Q: How do I rollback Artifactory configuration changes?
A: Use Artifactory configuration backup/restore, or revert changes via admin console. For repository layout changes, restore from configuration export.
Q: What logs should I inspect for troubleshooting?
A: Artifactory request logs, Docker daemon logs, Kubernetes pod logs, network connectivity logs, and CI/CD pipeline logs.
Q: How do I secure Artifactory credentials?
A: Use Kubernetes Secrets for imagePullSecrets, environment variables for CI/CD, API keys for automation, and integrate with secret management systems.
Q: How does Artifactory behave in production?
A: In production, Artifactory should be configured with HA, replication, HTTPS, proper security, monitoring, and backup. It handles higher load and provides enterprise-grade reliability.
Q: How do I improve Artifactory performance?
A: Implement CDN, use repository replication, optimize storage layout, increase resources, tune JVM settings, and implement caching for remote repositories.
Q: What are the common limitations of Artifactory?
A: Storage management complexity, initial setup learning curve, resource requirements for HA setup, and cost for enterprise features and support.
